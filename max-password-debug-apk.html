1. apa definisi debug?​

debug adalah sebuah metode yang di lakukan oleh para pemrograman dan pengembangan perangkat lunak utama mencari dan mengurangi bug, atau kerusakan di dalam sebuah komputer atau perangkat keras sehingga perangkat tersebut bekerja sesuai dengan harapan.


2. perbedaan BOOTCFG dan DEBUG


Perbedaan BOOTCFG dan DEBUG, untuk bisa menjawab pertanyaan tersebut mari kita bahas dahulu satu persatu kedua istilah tersebut. Kedua istilah tersebut merupakan istilah yang ada di pelajaran TI. BOOTCFG merupakan salah satu perintah yang terdapat pada sistem operasi windows. Perintah BOOTCFG sendiri memiliki fungsi untuk mengatur file yang tersembunyi yang bernama boot.ini. File tersebut berisi tentang dimana partisi windows berada.

Setelah mengetahui apa itu fungsi BOOTCFG, maka selanjutnya adalah perintah DEBUG. Perintah debug sendiri merupakan suatu perintah yang digunakan menjalankan suatu baris program. Debug sendiri biasanya digunakan oleh para pembuat aplikasi untuk mengetahui adanya kekurangan dalam aplikasi mereka. Dengan mengetahui kekurangan program yang dibuat maka para pengembang dapat segera memperbaikinya.

Demikian pembahasan mengenai perbedaan BOOTCFG dan Debug, kedua perintah tersebut memiliki pada perbedaan file yang dijalankan, apabila BOOTCFG hanya memodifikasi pada file boot.ini, sedangkan perintah Debug berfungsi untuk menjalankan suatu baris program untuk mengetahui kekurangan atau kesalahan pada program yang dibuat. 27. Jelaskan perbedaan mendasar antara BOOTCFG dengan DEBUG!


Perbedaan mendasar antara BOOTCFG dengan DEBUG terletak pada tugas dari masing-masing sintaks tersebut.

BOOTCFG adalah perintah utiliti yang digunakan dalam konsol Recovery pada sistem operasi berbasis Microsoft NT seperti Windows NT, Windows 2000, Windows XP dan Windows Server 2003. Perintah ini digunakan untuk membangun kembali (rebuild) atau menambahkan parameter ke file Boot.ini yang sudah ada, yang didalamnya terdapat pilihan sistem operasi pada saat proses startup dimana terdapat lebih dari satu sistem yang akan dijalankan sehingga user dapat memilih sistem operasi yang akan di boot setiap kali komputer dinyalakan. BOOTCFG sebagai perintah pada konsol Recvoery memiliki fungsi yang berbeda dengan perintah BOOTCFG.exe pada command prompt. BOOTCFG secara sederhana merupakan alat untuk mengedit khusus boot.ini, karena boot.ini masih dapat diedit menggunakan teks editor seperti Notepad. Beberapa pilihan pada BOOTCFG adalah

- /default - menyetel sistem operasi default pada saat proses boot

- /rebuild - menciptakan boot.ini dengan semua instalasinya secara otomatis

- /add - melakukan scan instalasi sistem operasi pada komputer dan memberikan wewenang ke user untuk menambahkan sistem operasi pada boot.ini

- /scan - mencari dan mengidentifikasi instalasi sistem operasi tetapi tidak mengubah boot.ini

- /list - memeriksa file boot.ini dan menampilkan semua entry dari sistem operasi yang ada

- /redirect - mengalihkan boot ke port spesifik

- /disableredirect - mematikan konfigurasi yang dilakukan oleh perintah redirect

DEBUG adalah sebuah perintah yang memungkinkan user untuk memeriksa dan mengubah isi memori sumber yang terjadi didalam sistem operasi. Perintah DEBUG memungkinkan user untuk memeriksa isi konten memori, membuat perubahan, dan menjalankan file COM, .exe dan file lainnya.
