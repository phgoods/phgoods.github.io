9. Jelaskan perbedaan antara single level directory, two level directory dan hierarchical


Jenis struktur direktori dalam sistem file:

Single level directory. Struktur direktori ini merupakan struktur direktori yang paling sederhana. Semua file disimpan dalam direktori yang sama. Two level directory (direktori dua tingkat). Direktori dua tingkat membuat direktori  terpisah untuk setiap pengguna, yang disebut Direktori File Pengguna (UFD). Saat pengguna masuk, root file dipanggil.Hierarchical merupakan suatu sistem Debian yang diorganisasikan mengikuti Filesystem Hierarchy Standard (FHS). Standar ini mendefinisikan tujuan masing-masing direktori.

Pembahasan

Beberapa sistem komputer menyimpan banyak file di disk, jadi Anda memerlukan struktur untuk mengatur data Anda untuk pengelolaan yang lebih mudah. Silberschatz, Galvin, dan Gagne mengklasifikasikan operasi pada direktori sebagai berikut:

Cari filenya. Menemukan entri untuk file tertentu dalam struktur direktori. File dengan nama  simbolik dan nama yang mirip menunjukkan koneksi antar file. Buat file. Anda perlu membuat file baru  dan menambahkannya ke  direktori.  Hapus file. Ketika Anda tidak lagi membutuhkan file, Anda harus menghapusnya dari direktori.  Menampilkan isi direktori. Menampilkan daftar file dalam direktori dan semua isi direktori dari file dalam daftar itu. Silahkan ganti nama filenya. Nama file mencerminkan isi file kepada pengguna. Oleh karena itu, nama file harus dapat diubah jika isi dan penggunaannya berubah atau menjadi tidak sesuai. Mengganti nama file dapat mengubah lokasinya dalam struktur direktori. [tex]\tt indexOf()[/tex] adalah method/function yang mengembalikan/menghasilkan indeks atau posisi pertama dari kemunculan suatu nilai teks atau karakter dalam string, dimulai dari indeks awal tertentu.
 

Pembahasan

Beberapa bahasa pemrograman menggunakan nama method [tex]\tt indexOf()[/tex], seperti Javascript dan Java, namun ada juga yang menggunakan nama method [tex]\tt index()[/tex], seperti pada Python.

Pada umumnya, [tex]\tt indexOf()[/tex] menerima 2 buah parameter, yaitu teks yang dicari dan indeks awal pencarian. Parameter kedua, yaitu indeks awal pencarian, bersifat opsional. Jika tidak diberikan, maka secara default indeks awal pencarian adalah posisi/urutan pertama dalam string.

Secara umum, sintaks dari method [tex]\tt indexOf()[/tex] adalah:

[tex]\begin{aligned}\bullet\ &\texttt{obj.indexOf(str)}\\\bullet\ &\texttt{obj.indexOf(str,\:indexAwal)}\end{aligned}[/tex]

Sintaks jenis pertama memiliki arti:
mencari indeks pertama yang ditemukan dari string [tex]\tt str[/tex] dalam string [tex]\tt obj[/tex]. Sedangkan sintaks jenis kedua memiliki arti:
mencari indeks pertama yang ditemukan dari string [tex]\tt str[/tex] dalam string [tex]\tt obj[/tex], dimulai dari posisi indeks [tex]\tt indexAwal[/tex] dalam string [tex]\tt obj[/tex].

Pada bahasa pemrograman Java, parameter pertama bisa berupa [tex]\tt int\ char[/tex], yang merepresentasikan sebuah karakter, atau suatu nilai Unicode tertentu.

Contoh penggunaannya (Java/Javascript) adalah sebagai berikut.

[tex]\texttt{teks = "hari ini adalah hari minggu"}[/tex]

[tex]\texttt{indeks = teks.indexOf("hari")}[/tex]
⇒ Nilai indeks = 0, karena kemunculan pertama dari string “hari” dalam variabel [tex]\tt teks[/tex] adalah pada posisi 0 (posisi awal).[tex]\texttt{indeks = teks.indexOf("hari", 4)}[/tex]
⇒ Nilai indeks = 16, karena kemunculan pertama dari string “hari” dalam variabel [tex]\tt teks[/tex] adalah pada posisi 16 (karakter ke-17 sampai 21).[tex]\texttt{indeks = teks.indexOf('i')}[/tex]
⇒ Nilai indeks = 1, karena kemunculan pertama dari karakter ’i’ dalam variabel [tex]\tt teks[/tex] adalah pada posisi 0 (posisi awal).

Perlu diperhatikan pula bahwa [tex]\tt indexOf()[/tex] bersifat case-sensitive. Jadi, jika paramter pertama pada contoh di atas adalah "Hari", maka [tex]\tt indexOf()[/tex] akan mengembalikan nilai -1, artinya tidak ada atau tak ditemukan.
